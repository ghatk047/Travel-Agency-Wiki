OB-SR-06

Price Comparison and Sort Logic Management

This process document outlines the management of price comparison and sort logic within the OTA Online Booking and Search domain, ensuring optimal user experience through personalized search results.

OB — OTA Online Booking and Search Search Ranking and Personalisation

Process Overview

Trigger User initiates a search query on Expedia Open World or Partner Central.
Outcome Users receive accurate, timely, and personalized search results based on their preferences and the current market conditions.
Systems Expedia Open World Partner Central Amadeus NDC Sabre GDS Travelport EVC One Key TravelAds AWS SageMaker Snowflake Kafka Salesforce Tableau

Risks & Exceptions

Complex query handling leading to increased processing times Integration issues with multiple GDS systems causing data discrepancies Inaccurate currency conversion affecting price comparisons Personalization models failing to adapt to diverse user profiles Cache management challenges impacting search performance User interface not aligning with user expectations Difficulty in capturing and analyzing user behavior Machine learning model overfitting leading to poor generalization Search algorithms becoming outdated as market conditions change Inaccurate reporting causing misinformed decision-making
